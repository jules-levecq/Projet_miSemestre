# Contribuer au projet Projet_miSemestre

Merci de contribuer ! Voici quelques règles et étapes pour faciliter les contributions.

## Processus de contribution
1. Forkez le dépôt et créez une branche feature/bugfix depuis `main`.
2. Travaillez sur votre branche et committez avec des messages clairs (ex: `feat: add autosave debounce`).
3. Ouvrez une Pull Request vers `main` décrivant les changements et les vérifications.

## Tests & Qualité
- Installez les dépendances frontend: `npm install`
- Frontend lint: `npm run lint` (si configuré)
- Backend tests: `mvn test`

## Style de commit
Utilisez des messages de type Conventional Commits simples :
- `feat: ` pour les nouvelles fonctionnalités
- `fix: ` pour les corrections
- `chore: ` pour les tâches de maintenance

## Revue
- Incluez des captures d'écran pour les changements UI
- Documentez les nouvelles fonctions dans `README.md` et `docs/`
